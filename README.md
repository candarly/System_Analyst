# Добавление функционала в личный кабинет клиента

Проект был посвящён разработке новой функциональности — возможности оставлять отзыв в личном кабинете клиента. Работа выполнялась полностью самостоятельно: от анализа требований до визуального прототипа. Были описаны пользовательские истории и сценарии использования, сформирована модель данных, разработана последовательность работы системы в виде sequence-диаграммы, а также создан прототип интерфейса будущего функционала.

## Используемые инструменты и артефакты

* User Stories (описание пользовательских потребностей и целей)
* Use Case (сценарии использования с альтернативами и потоками событий)
* Моделирование данных (ER-диаграмма / сущности и связи)
* Sequence Diagram (взаимодействие пользователя, фронтенда и бэкенда, plantuml)
* Прототип интерфейса (Drawio)

## Чему удалось научиться

* превращать пользовательские запросы в формализованные требования;
* описывать логику поведения системы через use cases и последовательности взаимодействий;
* строить корректную модель данных под новую функциональность;
* визуализировать процессы через UML-диаграммы;
* проектировать интерфейс, учитывая UX и поток пользователя;
* связывать бизнес-цель, требования, модель данных и интерфейс в единое решение.
